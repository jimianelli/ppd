Posterior Predictive Reference Point Estimation in an Age-Structured Model

Overview

This document describes a workflow to propagate uncertainty in a statistical age-structured stock assessment model with time-varying selectivity into derived reference points such as \(F_{35\%}\) and \(F_{40\%}\).

The approach combines posterior predictive simulation with re-estimation.

Browse the repository on GitHub: jimianelli/ppd

Model fitting

We fit the model using RTMB:

obj <- MakeADFun(data, parameters, DLL="your_model", random="states")
opt <- nlminb(obj$par, obj$fn, obj$gr)
rep <- sdreport(obj)

Posterior uncertainty is approximated using:

  • Multivariate normal approximation from the joint precision matrix, or
  • MCMC sampling (preferred)

Posterior predictive simulation

For each posterior draw \(\theta^{(s)}\), we simulate data:

\[ y^{rep,(s)} \sim p(y \mid \theta^{(s)}) \]

This is implemented using the SIMULATE block in the TMB model.

Re-estimation

Each simulated dataset is refit:

obj_sim <- MakeADFun(sim_data, parameters, DLL="your_model", random="states")
opt <- nlminb(obj_sim$par, obj_sim$fn, obj_sim$gr)
rep <- sdreport(obj_sim)

This step is critical because it propagates estimation uncertainty in selectivity.

Spawning potential ratio (SPR)

SPR is computed as:

\[ SPR(F) = \frac{SSB(F)}{SSB(F=0)} \]

where spawning biomass per recruit is derived from survivorship under fishing mortality \(F\).

Estimation of SPR-Based Reference Points

For each refit, we solve for quantities such as:

\[ SPR(F_{35\%}) = 0.35 \]

and, similarly, \(SPR(F_{40\%}) = 0.40\) using a root-finding algorithm.

Algorithm

for (s in 1:S) {
  theta_s <- draw_theta()

  sim_data <- simulate_data(theta_s)

  fit_s <- refit_model(sim_data)

  sel_hat <- extract_selectivity(fit_s$rep)

  F35[s] <- find_F35(sel_hat, M, wa, mat)
  F40[s] <- find_F40(sel_hat, M, wa, mat)
}

Output

The result is a posterior predictive distribution for each reference point, for example:

\[ \{ F_{35\%}^{(s)} \}_{s=1}^S \]

and likewise \(\{ F_{40\%}^{(s)} \}_{s=1}^S\).

This reflects:

  • Parameter uncertainty
  • Observation error
  • Re-estimation variability
  • Time-varying selectivity uncertainty

Management Application

One motivation for computing the posterior predictive distribution of quantities such as \(F_{35\%}\) and \(F_{40\%}\) is to support decisions when a risk table score is elevated for assessment-related considerations or population dynamics considerations.

When the risk table remains at a normal level, a single plug-in estimate of a reference point may be adequate for routine advice. When concern is raised, however, the distribution itself becomes informative because it shows how much uncertainty there is in the appropriate SPR-based target once observation error, parameter uncertainty, and refitting instability are propagated.

In that setting, the PPD provides a defensible way to translate increased concern into more precautionary advice. For example, one could compare the distributions of \(F_{35\%}\) and \(F_{40\%}\), use lower quantiles of the estimated reference point distribution rather than the median, or otherwise scale the applied harvest rate according to whether the risk table score indicates moderate or extreme concern. The main point is that an elevated risk score can be tied to a quantitative change in how conservative the chosen fishing mortality target should be, rather than relying only on an ad hoc adjustment.

Notes

  • Selectivity used for SPR must match the operational definition (e.g., terminal year, block average)
  • Recruitment assumptions can influence results if weight-at-age or maturity vary over time
  • The same workflow applies to \(F_{35\%}\), \(F_{40\%}\), or other SPR-based targets
  • Flexible selectivity increases variance in these derived reference points

Extensions

  • Compare plug-in vs re-estimated distributions
  • Evaluate bias via median(\(F_{35\%}\)) or median(\(F_{40\%}\)) vs true generating value
  • Use this framework to formalize risk-table adjustments to target fishing mortality
  • Use this framework inside MSE loops
